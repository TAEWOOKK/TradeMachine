# 매매 전략 규칙서

> TradeMachine의 핵심인 규칙 기반(Rule-based) 스윙 트레이딩 전략을 정의한 문서.
> 모든 매매는 이 문서의 조건에 따라 기계적으로 실행된다.

---

## 1. 전략 개요

| 항목 | 값 |
|------|-----|
| 전략 유형 | 규칙 기반 스윙 트레이딩 |
| 대상 시장 | 국내주식 (코스피/코스닥) |
| 보유 기간 | 수일 ~ 수주 (단타 X) |
| 핵심 지표 | 이동평균선 (MA: Moving Average) |
| 판단 주기 | 스케줄러 간격 (기본 5분) |
| 소수점 매매 | 사용하지 않음 (정수 단위만) |

---

## 2. 핵심 지표: 이동평균선 (MA)

### 2.1 사용하는 이동평균선

| 이동평균선 | 기간 | 역할 |
|-----------|------|------|
| **단기 MA** | 5일 | 최근 추세 반영. 빠르게 반응 |
| **중기 MA** | 20일 | 중기 추세. 매매 신호의 기준선 |
| **장기 MA** | 60일 | 장기 추세 판별. 필터 역할 (추후 확장) |

### 2.2 계산 방법

단순이동평균(SMA: Simple Moving Average) 사용:

```
MA(N) = (최근 N일간 종가의 합) / N

예시: MA(5)
  = (오늘 종가 + 어제 종가 + ... + 4일전 종가) / 5
```

### 2.3 데이터 소스

- KIS API의 **기간별 시세 조회** (`FHKST03010100`) 사용
- 최소 60일치 일봉 데이터를 조회하여 MA 계산
- 매 스케줄링 주기마다 최신 데이터로 재계산

## 3. 매수 조건 (진입 규칙)

### 3.1 골든 크로스 매수 (신호 확인 기간 포함)

> 단기 이동평균선이 중기 이동평균선을 **아래에서 위로 돌파**한 후,
> **일정 기간 동안 유지**되는 것을 확인한 뒤 매수한다.
> 하루만 보고 매수하지 않는다.

#### 왜 신호 확인이 필요한가?

골든크로스는 하루 발생 후 다음날 바로 꺾이는 "가짜 신호(Whipsaw)"가 자주 나타난다.
며칠간 유지되는지 확인해야 진짜 추세 전환인지 판별할 수 있다.

```
가짜 신호 예시:                     진짜 신호 예시:

MA(5) ──╲  ╱── 하루만 올라갔다 꺾임    MA(5) ──╲  ╱──────── 계속 위에 유지
MA(20) ───╳───                       MA(20) ───╳───
          ↑ 이런건 무시해야 함                    ↑ 이런게 진짜 매수 타이밍
```

#### 조건식 (2단계 확인)

```
[1단계: 골든크로스 감지]
  - 최근 N일(SIGNAL_LOOKBACK_DAYS) 내에 교차가 발생했는가?
  - 즉, N일 전에는 MA(5) < MA(20)이었는데
    이후 MA(5) >= MA(20)으로 전환된 시점이 있는가?

[2단계: 신호 확인 (유지 기간 검증)]
  - 교차 발생 이후, 연속 M일(SIGNAL_CONFIRM_DAYS) 동안
    MA(5) > MA(20) 상태가 유지되고 있는가?

[3단계: 추세 건전성 확인]
  - 현재가 > MA(20)              ← 중기선 위에 있을 것
  - MA(20) 기울기 > 0            ← 중기선 자체가 상승 추세
    (오늘 MA(20) > 3일전 MA(20))

[요약]
  교차 발생 + M일 연속 유지 + 현재가 > MA(20) + 중기선 상승
  → 이 모든 조건이 충족되어야 매수
```

| 설정 | 기본값 | 설명 |
|------|--------|------|
| `SIGNAL_LOOKBACK_DAYS` | 14일 | 최근 며칠 내 교차 발생을 감지할 범위 |
| `SIGNAL_CONFIRM_DAYS` | 3일 | 교차 후 연속 유지 확인 일수 |

#### 구체적 예시

```
                     5일선    20일선    상태
  14일전              67,000   70,500   5일선 < 20일선
  ...
  8일전               68,500   70,100   5일선 < 20일선
  7일전 (월)          68,000   70,000   5일선 < 20일선
  6일전 (화)          69,000   70,000   5일선 < 20일선
  5일전 (수)          70,200   70,000   ★ 교차 발생 (5일선이 위로)
  4일전 (목)          70,800   70,100   유지 1일차 ✓
  3일전 (금)          71,500   70,200   유지 2일차 ✓
  2일전 (월)          72,000   70,400   유지 3일차 ✓ → 확인 완료!
  1일전 (화)          72,300   70,600   계속 유지 중
  오늘   (수)         72,500   70,800   → ★ 매수 실행 ★

  현재가: 73,000원 > MA(20) 70,800원 ✓
  MA(20) 기울기: 70,800 > 70,200(3일전) → 상승 추세 ✓
```

#### 판별 플로우

```
일봉 데이터 조회 (최근 60일)
  │
  ▼
최근 14일간 MA(5), MA(20) 일별 계산
  │
  ▼
(1) 최근 14일 내에 골든크로스 교차 지점이 있는가?
    (MA(5)가 MA(20) 아래 → 위로 전환된 날이 있는가?)
    │
    ├─ NO → 매수 신호 아님 (SKIP)
    │
    └─ YES
         │
         ▼
(2) 교차 이후 연속 3일간 MA(5) > MA(20) 유지 중인가?
    │
    ├─ NO → 아직 확인 안됨 (SKIP, 다음 스캔에서 재확인)
    │
    └─ YES
         │
         ▼
(3) 현재가 > MA(20)?
    │
    ├─ NO → 가격이 중기선 아래 (SKIP)
    │
    └─ YES
         │
         ▼
(4) MA(20) 상승 추세? (오늘 MA(20) > 3일전 MA(20))
    │
    ├─ NO → 하락 추세에서의 반등일 수 있음 (SKIP)
    │
    └─ YES
         │
         ▼
(5) 거래량 확인 (Volume Confirmation)
    골든크로스 교차일의 거래량 > MA(20) 거래량의 1.5배?
    │
    ├─ NO → 거래량 미수반 교차 (신뢰도 낮음, SKIP)
    │
    └─ YES
         │
         ▼
(6) RSI 과열 필터
    30 <= RSI(14) <= 70?
    │
    ├─ RSI > 70 → 과매수 상태 (SKIP "이미 과열")
    ├─ RSI < 30 → 과매도 하락 추세 (SKIP "추세 약화")
    │
    └─ 30~70 → ★ 매수 신호 확정 ★
```

### 3.2 보조 지표: 거래량 확인 (Volume Confirmation)

> 골든크로스가 발생해도 **거래량이 수반되지 않으면** 기관/외국인의 참여가 없는
> 허약한 신호일 가능성이 높다. 거래량이 평소보다 1.5배 이상 증가한 교차만 신뢰한다.

**조건식:**

```
교차일 거래량 > MA(20) 평균 거래량 × VOLUME_CONFIRM_RATIO (1.5)
```

| 설정 | 기본값 | 설명 |
|------|--------|------|
| `VOLUME_CONFIRM_RATIO` | `1.5` | 교차일 거래량이 평균의 N배 이상이어야 유효 |

**왜 효과적인가:**

```
거래량 미수반 골든크로스:
  → 소수의 매수세로 일시적 교차 → 곧 되돌림 → 손절 가능성 높음
  → 승률 ~35%

거래량 수반 골든크로스:
  → 기관/외국인 참여 가능성 → 추세 지속력 높음
  → 승률 ~50-55% (10-15%p 향상)
```

### 3.3 보조 지표: RSI 과열 필터

> RSI(Relative Strength Index, 14일)를 사용하여
> **이미 과열된 종목**에 뒤늦게 진입하는 것을 방지한다.

**RSI 계산:**

```
RS = (최근 14일 평균 상승폭) / (최근 14일 평균 하락폭)
RSI = 100 - (100 / (1 + RS))
```

**필터 조건:**

| RSI 범위 | 판정 | 행동 |
|----------|------|------|
| RSI > 70 | 과매수 (Overbought) | 매수 SKIP — 이미 고점 근처 |
| RSI < 30 | 과매도 (Oversold) | 매수 SKIP — 하락 추세 강함 |
| 30 ≤ RSI ≤ 70 | 적정 범위 | 매수 허용 |

| 설정 | 기본값 | 설명 |
|------|--------|------|
| `RSI_PERIOD` | `14` | RSI 계산 기간 (일) |
| `RSI_OVERBOUGHT` | `70` | 과매수 기준 |
| `RSI_OVERSOLD` | `30` | 과매도 기준 |

### 3.4 상승추세 진입 매수 (UPTREND_ENTRY)

> 골든크로스 lookback(14일) 기간 밖에서 교차가 발생했지만,
> **현재 이미 상승추세에 있는 종목**을 감지하여 진입하는 보조 매수 전략이다.
> 골든크로스 조건을 충족하지 못하는 종목에 대해서만 평가한다.

#### 왜 필요한가?

```
골든크로스가 20일 전에 발생한 종목:
  → SIGNAL_LOOKBACK_DAYS(14일) 범위 밖이라 골든크로스 매수 신호 없음
  → 하지만 MA(5) > MA(20)이 계속 유지되고 있고, 건강한 상승추세
  → 이런 종목을 놓치지 않기 위한 보완 전략
```

#### 조건식 (모두 충족 시 매수)

```
[1] MA(5) > MA(20)이 최근 5일 연속 유지
[2] 현재가 > MA(20)
[3] MA(20)이 상승 추세 (오늘 MA(20) > 3일전 MA(20))
[4] MA(5)와 MA(20)의 괴리율 5% 이내
    → (MA(5) - MA(20)) / MA(20) × 100 <= 5%
    → 괴리율이 너무 크면 과열 구간이므로 진입 위험
[5] 최근 5일 중 거래량이 20일 평균 이상인 날이 2일 이상
    → 거래량이 지지하는 상승추세만 진입
```

> **참고:** UPTREND_ENTRY는 골든크로스 판별에서 SKIP된 종목에 대해서만 추가 평가한다.
> 골든크로스 조건을 이미 충족한 종목은 GOLDEN_CROSS로 매수된다.

### 3.5 매수 수량 결정

```
1. 매수 가능 금액 조회 (KIS API)
2. 종목당 최대 투자 금액 = 매수 가능 금액 × MAX_INVESTMENT_RATIO
3. 매수 수량 = 종목당 최대 투자 금액 ÷ 현재가 (소수점 버림)
4. 매수 수량이 0이면 매수하지 않음
```

**예시:**
```
매수 가능 금액: 10,000,000원
MAX_INVESTMENT_RATIO: 0.1 (10%)
종목당 최대 투자 금액: 1,000,000원
현재가: 70,000원
매수 수량: 1,000,000 ÷ 70,000 = 14주 (소수점 버림)
```

### 3.6 매수 주문 방식

| 항목 | 값 | 이유 |
|------|-----|------|
| 주문 유형 | **시장가** (`ORD_DVSN: "01"`) | 스윙 트레이딩은 즉시 체결이 중요 |
| 대안 | 지정가 (설정으로 전환 가능) | 슬리피지 관리가 필요한 경우 |

## 4. 매도 조건 (청산 규칙)

매도는 5가지 조건 중 **하나라도 충족**되면 우선순위 순으로 실행한다.

### 4.1 데드 크로스 매도 (전략 신호)

> 단기 이동평균선이 중기 이동평균선을 **위에서 아래로 돌파**하고,
> **2일 연속 유지**되면 매도한다.
>
> 단, 매도 확인 기간은 매수보다 짧게 설정한다 (하락은 빠르게 대응).

**조건식:**

```
[1단계: 데드크로스 감지]
  - 최근 5일 내에 MA(5)가 MA(20) 위 → 아래로 전환된 시점이 있는가?

[2단계: 신호 확인]
  - 교차 이후, 연속 2일(SELL_CONFIRM_DAYS) 동안
    MA(5) < MA(20) 상태가 유지되고 있는가?

→ 두 조건 모두 충족 시 매도
```

| 설정 | 기본값 | 설명 |
|------|--------|------|
| `SELL_CONFIRM_DAYS` | 2일 | 매수 확인(3일)보다 짧게. 하락엔 빠르게 대응 |

> **참고:** 손절(-5%)은 확인 기간 없이 즉시 실행한다. 데드크로스 확인을 기다리다 손실이 커지는 것을 방지.

### 4.2 손절 매도 (리스크 관리)

> 매입가 대비 일정 비율 이상 하락하면 **무조건 매도**한다.

**조건식:**

```
현재 수익률 = (현재가 - 매입평균가) / 매입평균가

현재 수익률 <= STOP_LOSS_RATE 이면 → 즉시 매도
```

| 설정 | 기본값 | 설명 |
|------|--------|------|
| `STOP_LOSS_RATE` | `-5.0` (%) | 매입가 대비 5% 하락 시 손절 |

**예시:**
```
매입평균가: 70,000원
손절 라인: 70,000 × (1 - 0.05) = 66,500원
현재가 66,000원 → 수익률 -5.7% → 손절 조건 충족 → 매도
```

### 4.3 익절 매도 (수익 실현)

> 매입가 대비 일정 비율 이상 상승하면 **수익 실현 매도**한다.

**조건식:**

```
현재 수익률 >= TAKE_PROFIT_RATE 이면 → 매도
```

| 설정 | 기본값 | 설명 |
|------|--------|------|
| `TAKE_PROFIT_RATE` | `15.0` (%) | 매입가 대비 15% 상승 시 익절 |

### 4.4 트레일링 스탑 매도 (수익 보호)

> 일정 수익에 도달한 후, 고점 대비 일정 비율 하락하면 매도한다.
> 익절 라인(+15%)에 도달하지 못해도 수익을 보호할 수 있다.

#### 왜 필요한가?

```
트레일링 스탑 없이:
  매수 70,000원 → 최고점 80,000원(+14.3%) → 하락 71,000원(+1.4%)
  → 익절 15%에 못 미쳐서 매도 안 함. 손절 -5%에도 안 걸림.
  → 결국 이익 대부분을 반납.

트레일링 스탑 적용:
  매수 70,000원 → 최고점 80,000원(+14.3%) → 활성화 (8% 초과)
  → 기준가 = 80,000 × (1 - 0.04) = 76,800원
  → 현재가 76,000원 < 76,800원 → 매도! 수익 +8.6% 확보.
```

**조건식 (2단계):**

```
[1단계: 활성화]
  보유 중 현재 수익률이 TRAILING_STOP_ACTIVATE(+8%) 이상 도달한 적이 있는가?
  → 매 스캔마다 종목별 최고가를 메모리에 갱신

[2단계: 매도 판단]
  현재가 <= 최고가 × (1 - TRAILING_STOP_RATE / 100) 이면 → 매도
```

| 설정 | 기본값 | 설명 |
|------|--------|------|
| `TRAILING_STOP_ACTIVATE` | `8.0` (%) | 수익률 +8% 이상 시 트레일링 활성화 |
| `TRAILING_STOP_RATE` | `4.0` (%) | 활성화 후 고점 대비 4% 하락 시 매도 |

> 구현: TradingService에 `_highest_prices: dict[str, int]`를 관리.
> 매 스캔 시 현재가와 비교하여 갱신. 앱 재시작 시 소실되나 스캔 반복으로 자연 복구.

### 4.5 보유 기간 초과 매도 (자금 효율)

> 매수 후 일정 영업일이 지나도 익절/손절/데드크로스에 해당하지 않으면 강제 매도한다.
> 횡보 종목이 자금을 무기한 묶는 것을 방지한다.

**조건식:**

```
현재 영업일 - 최초 매수일 > MAX_HOLDING_DAYS → 매도
```

| 설정 | 기본값 | 설명 |
|------|--------|------|
| `MAX_HOLDING_DAYS` | `20` | 영업일 기준 약 1개월 초과 시 청산 |

> 구현: orders 테이블에서 해당 종목의 가장 최근 BUY 성공 기록의 날짜를 조회.

### 4.6 매도 판별 플로우 (우선순위 순서)

```
보유 종목 순회
  │
  ▼
┌─ 종목별 체크 ──────────────────────────────┐
│                                              │
│  (1) 손절 체크 (최우선, P0)                   │
│      수익률 <= -5%?                          │
│      ├─ YES → ★ 즉시 매도 ★                 │
│      └─ NO ↓                                │
│                                              │
│  (2) 익절 체크 (P1)                           │
│      수익률 >= +15%?                         │
│      ├─ YES → ★ 매도 ★                      │
│      └─ NO ↓                                │
│                                              │
│  (3) 트레일링 스탑 (P2)                       │
│      활성화 상태 + 고점 대비 4% 하락?          │
│      ├─ YES → ★ 매도 ★                      │
│      └─ NO ↓                                │
│                                              │
│  (4) 보유 기간 초과 (P3)                      │
│      보유 영업일 > 20일?                      │
│      ├─ YES → ★ 매도 ★                      │
│      └─ NO ↓                                │
│                                              │
│  (5) 데드 크로스 체크 (P4)                    │
│      MA(5) 하향 돌파 MA(20) + 2일 유지?       │
│      ├─ YES → ★ 매도 ★                      │
│      └─ NO → 보유 유지 (HOLD)                │
│                                              │
└──────────────────────────────────────────────┘
```

### 4.7 매도 주문 방식

| 항목 | 값 | 이유 |
|------|-----|------|
| 주문 유형 | **시장가** (`ORD_DVSN: "01"`) | 손절 시 즉시 체결이 최우선 |
| 수량 | **전량 매도** | 스윙 트레이딩은 분할 매도보다 단순 청산 우선 |

## 5. 리스크 관리 규칙

### 5.1 포지션 관리

| 규칙 | 값 | 설명 |
|------|-----|------|
| 종목당 최대 투자 비율 | `MAX_INVESTMENT_RATIO` = 10% | 한 종목에 전체 자산의 10% 초과 투자 금지 |
| 최대 동시 보유 종목 수 | `MAX_HOLDING_COUNT` = 5 | 분산 투자. 동시에 5종목까지만 보유 |
| 손절 라인 | `STOP_LOSS_RATE` = -5% | 매입가 대비 5% 하락 시 무조건 손절 |
| 익절 라인 | `TAKE_PROFIT_RATE` = +15% | 매입가 대비 15% 상승 시 수익 실현 |

### 5.2 매수 전 체크리스트 (Service 레이어)

매수 신호가 발생해도 아래 조건을 **모두 통과**해야 실제 주문이 나간다:

```
매수 신호 발생
  │
  ▼
(1) 이미 보유 중인 종목인가?
    ├─ YES → 매수 SKIP (중복 매수 방지)
    └─ NO ↓

(2) 현재 보유 종목 수 >= MAX_HOLDING_COUNT?
    ├─ YES → 매수 SKIP (최대 보유 수 초과)
    └─ NO ↓

(3) 매수 가능 금액 조회
    투자 금액 = 매수 가능 금액 × MAX_INVESTMENT_RATIO
    매수 수량 = 투자 금액 ÷ 현재가
    ├─ 수량 == 0 → 매수 SKIP (잔액 부족)
    └─ 수량 >= 1 ↓

(4) 매수 주문 실행
```

### 5.3 하루 매매 횟수 제한

| 규칙 | 값 | 설명 |
|------|-----|------|
| 일일 최대 매수 횟수 | `MAX_DAILY_BUY_COUNT` = 3회 | 하루에 3번 이상 매수하지 않음 |
| 일일 최대 매도 횟수 | 제한 없음 | 손절은 제한 없이 즉시 실행 |

> 카운트는 `run_pre_market()`에서 0으로 초기화. 매수 성공 시 +1 증가.

### 5.4 재매수 쿨다운

> 동일 종목을 매도한 직후 다시 매수하면 불필요한 수수료가 발생하고,
> 데드크로스 매도 후 아직 살아 있는 골든크로스 확인 기간에 의해 즉시 재진입하는 문제를 방지한다.

| 규칙 | 값 | 설명 |
|------|-----|------|
| 재매수 금지 시간 | `REBUY_COOLDOWN_HOURS` = 24시간 | 매도 후 24시간 이내 동일 종목 매수 금지 |

```
매수 판단 시:
  orders 테이블에서 해당 종목의 최근 SELL 성공 기록 조회
  → 현재 시각 - 매도 시각 < 24시간 → 매수 SKIP "재매수 쿨다운"
```

**단, 아래 6.3의 종목 품질 필터를 통과하는 종목만 실제 매수 대상이 된다.**

### 6.3 종목 품질 필터 (잡주 방지)

> 골든크로스가 발생해도, 종목 자체의 품질이 일정 수준 이상이어야 매수한다.
> 잡주는 거래량이 적어 소수의 매매로 MA가 쉽게 왜곡되고,
> 세력에 의한 가짜 골든크로스가 빈번하며, 상장폐지 등 치명적 위험이 있다.

#### 왜 필터링이 필요한가?

```
잡주의 위험:
  - 일 거래량 1,000주인 종목에서 누군가 2,000주 매수
    → 가격 급등 → MA(5) 왜곡 → 가짜 골든크로스 발생
    → 봇이 매수 → 다음날 급락 → 손실

  - 시가총액 100억 미만 종목
    → 작전세력 개입 가능성 높음
    → 상장폐지 위험
    → 유동성 부족으로 매도 시 큰 슬리피지
```

#### 필터 조건

관심 종목에 등록되어 있더라도, **매수 시점에** 아래 조건을 모두 통과해야 실제 주문이 나간다:

```
[필수 필터 — 하나라도 미달 시 매수 SKIP]

(1) 주가 필터
    현재가 >= MIN_STOCK_PRICE (기본: 5,000원)
    → 1,000원짜리 동전주 방지

(2) 거래대금 필터
    당일 누적 거래대금 >= MIN_TRADING_VALUE (기본: 10억원)
    → 유동성이 충분한 종목만 매매
    → 거래대금 = 거래량 × 평균가. KIS API 현재가 시세의 acml_tr_pbmn 필드

(3) 거래량 필터
    당일 거래량 >= MIN_TRADING_VOLUME (기본: 50,000주)
    → 거래량이 너무 적으면 시장가 주문 시 불리한 가격에 체결

(4) 시가총액 필터
    시가총액 >= MIN_MARKET_CAP (기본: 5,000억원)
    → 중대형주 이상만 매매
    → KIS API 현재가 시세의 hts_avls 필드 사용

    ※ hts_avls 단위 주의:
      KIS API의 hts_avls(HTS 시가총액)는 **"억원"** 단위로 반환된다.
      코드에서 hts_avls × 100,000,000 하여 **"원"** 단위로 변환 후
      MIN_MARKET_CAP(500,000,000,000원 = 5,000억원)과 비교한다.
```

#### 필터 적용 시점과 위치

> 필터는 골든크로스 판별 **전에** 적용한다. (API 호출 절약)
> 중복/한도 체크 → 시장 필터 → 안전성 필터 → 품질 필터 → 재매수 쿨다운 → 골든크로스/상승추세 판별 → 매수

```
관심 종목 순회
  │
  ▼
[중복/한도 체크]
  │
  ├─ 이미 보유 중? → SKIP
  ├─ 보유 종목 수 >= 5? → SKIP
  ├─ 일일 매수 횟수 >= 3? → SKIP
  │
  ▼ (모두 통과)
[시장 필터] ← run_scan()에서 1회 체크한 market_ok 플래그 사용
  │
  ├─ KOSPI < MA(20)? → SKIP "하락장 매수 보류"
  │
  ▼ (통과)
[안전성 필터] ← 관리종목, 투자유의, 거래정지 등 제외
  │
  ▼
[종목 품질 필터] ← 잡주 제외
  │
  ├─ 현재가 < 5,000원? → SKIP "저가주 제외"
  ├─ 거래대금 < 10억? → SKIP "유동성 부족"
  ├─ 거래량 < 5만주? → SKIP "거래량 부족"
  ├─ 시가총액 < 5,000억? → SKIP "소형주 제외"
  │
  ▼ (모두 통과)
[재매수 쿨다운]
  │
  ├─ 24시간 내 매도 이력? → SKIP "재매수 쿨다운"
  │
  ▼ (통과)
[골든크로스 판별] ← 일봉 API 호출은 여기서만
  │
  ├─ 골든크로스 4단계 확인 → 충족 시 GOLDEN_CROSS 매수
  │
  ▼ (골든크로스 미충족 시)
[상승추세 판별 (UPTREND_ENTRY)]
  │
  ├─ 5일 연속 MA5 > MA20 + 현재가 > MA20 + MA20 상승
  │   + 괴리율 5% 이내 + 거래량 조건 → 충족 시 UPTREND_ENTRY 매수
  ├─ 미충족 → SKIP
  │
  ▼ (확인 완료)
[매수 수량 계산 + 주문 실행]
```

#### 필터 기준값 가이드

| 필터 | 보수적 | 기본값 (권장) | 공격적 |
|------|--------|-------------|--------|
| 최소 주가 | 10,000원 | 5,000원 | 2,000원 |
| 최소 거래대금 | 50억원 | 10억원 | 5억원 |
| 최소 거래량 | 100,000주 | 50,000주 | 10,000주 |
| 최소 시가총액 | 1조원 | 5,000억원 | 1,000억원 |

> **보수적**: 대형 우량주 위주. 안정적이나 수익 기회 제한.
> **기본값**: 중대형주. 유동성과 안정성의 균형.
> **공격적**: 중소형주 포함. 수익 기회 많으나 리스크 증가.

#### 설정값

> [project-plan.md §7 설정 관리](./project-plan.md) 참조.
> 주요: `MIN_STOCK_PRICE`(5,000원), `MIN_TRADING_VALUE`(10억원), `MIN_TRADING_VOLUME`(50,000주)

#### 필터 데이터 소스 (KIS API 매핑)

| 필터 | KIS API 필드 | API |
|------|-------------|-----|
| 현재가 | `stck_prpr` | 주식현재가 시세 (`FHKST01010100`) |
| 누적 거래대금 | `acml_tr_pbmn` | 주식현재가 시세 (`FHKST01010100`) |
| 누적 거래량 | `acml_vol` | 주식현재가 시세 (`FHKST01010100`) |
| 시가총액 | `hts_avls` (억원 단위, ×1억 변환 필요) | 주식현재가 시세 (`FHKST01010100`) |

> 현재가 시세 API 한 번 호출로 주가, 거래대금, 거래량을 모두 얻을 수 있다.
> 추가 API 호출 없이 필터링 가능.

### 9.1 시장 이벤트 예외

#### 상한가/하한가

| 상황 | 문제 | 대응 |
|------|------|------|
| 보유 종목이 하한가 | 매도 주문 넣어도 **체결 불가** (매수자 없음) | 매도 주문 후 미체결 상태 확인. 다음 스캔에서 재시도. 로그 기록 |
| 관심 종목이 상한가 | 매수 주문 넣어도 **체결 불가** (매도자 없음) | 상한가 종목은 매수 SKIP. 이미 추세를 놓친 것 |
| 상한가/하한가 감지 | 현재가 == 상한가 or 하한가 | KIS API 응답의 상한가/하한가 필드로 판별 가능 |

**규칙:**
```
매수 시: 현재가 == 상한가 → 매수 SKIP
매도 시: 시장가로 주문. 미체결이면 다음 스캔에서 재주문
```

#### 갭 하락 (손절라인 초과)

> 전일 종가 70,000원 → 오늘 시가 60,000원으로 시작하는 경우
> 손절라인(-5%, 66,500원)을 한번에 뛰어넘어 -14% 손실 상태로 시작

| 상황 | 문제 | 대응 |
|------|------|------|
| 갭 하락으로 손절라인 초과 | 이미 손절라인을 넘어선 상태 | **즉시 시장가 매도**. 더 기다리면 손실만 커짐 |
| 갭 상승 | 이동평균선이 급변동 | 이미 MA 계산에 반영되므로 별도 처리 불필요 |

**규칙:**
```
손절 체크는 "수익률 <= STOP_LOSS_RATE" 이므로
-5%든 -15%든 조건만 충족되면 즉시 매도 실행.
갭 하락이든 점진 하락이든 동일하게 처리.
```

#### 서킷브레이커 (Circuit Breaker) / 사이드카

> 코스피/코스닥 지수가 급락하면 시장 전체 거래가 중단된다.

| 종류 | 발동 조건 | 거래 중단 시간 |
|------|----------|--------------|
| 사이드카 | 선물가격 5% 이상 변동 1분 지속 | 프로그램매매 5분 중단 |
| 서킷브레이커 1단계 | 지수 8% 이상 하락 1분 지속 | 20분 거래 중단 |
| 서킷브레이커 2단계 | 지수 15% 이상 하락 1분 지속 | 20분 거래 중단 |
| 서킷브레이커 3단계 | 지수 20% 이상 하락 1분 지속 | 잔여 시간 거래 중단 |

**대응:**
```
- API 호출 시 주문 거부 응답이 오면 → 로그 기록 + 재시도 하지 않음
- 스케줄러는 계속 동작하되, 주문 실패 시 다음 스캔까지 대기
- 서킷브레이커 해제 후 자동으로 정상 스캔 재개
- 서킷브레이커 발동 시 새로운 매수는 금지 (패닉 매수 방지)
```

#### 거래정지 종목

> 관리종목 지정, 투자주의, 상장폐지 심사 등으로 거래가 정지된 종목

**대응:**
```
- 시세 조회 시 비정상 응답 (거래량 0, 가격 0 등) → 해당 종목 SKIP
- 보유 중인 종목이 거래정지 → 매도 불가. 로그 기록 + 알림
- 관심 종목에서 수동 제거 필요
```

#### 배당락일

> 배당 기준일 다음 영업일에 배당금만큼 주가가 자연 하락한다.
> 이것은 실제 하락이 아니라 배당금 반영일 뿐인데, MA가 꺾이면서 가짜 매도 신호 발생 가능.

**문제 예시:**
```
배당 전: 70,000원 → 배당락: 68,000원 (배당금 2,000원 반영)
실제 가치는 동일하지만 MA(5)가 급락하면서 데드크로스 발생
→ 불필요한 매도 신호
```

**대응 (Phase 5 이후):**
```
- 초기에는 별도 처리 없음 (스윙 트레이딩은 며칠 후 회복되는 경우가 많음)
- SELL_CONFIRM_DAYS=2로 확인 기간이 있으므로 1일 반등하면 매도 안 함
- 추후 배당락일 캘린더 데이터 연동하여 배당락일 전후 매도 판단 보류 가능
```

#### 주식 분할/병합 (액면분할 등)

> 삼성전자가 50:1 액면분할하면 250만원 → 5만원으로 가격이 급변.
> 과거 일봉 데이터도 소급 적용되지만, 분할 직후 API 데이터가 불안정할 수 있음.

**대응:**
```
- MA 계산용 일봉 데이터는 KIS API에서 수정주가(adjusted)를 제공하므로 대부분 자동 반영
- 분할/병합 당일에는 비정상 MA 값이 나올 수 있음 → 전일 대비 가격 변동률이
  비정상적으로 큰 경우 (예: |변동률| > 30%) 해당 종목 매매 보류
```

**규칙:**
```
전일 대비 등락률 절대값 > ABNORMAL_CHANGE_RATE(30%) → 매매 보류
(상한가 30%, 하한가 30%를 초과하는 변동은 비정상)
```

#### 공휴일 / 주말 / 휴장일 / 반일 장

| **법정 공휴일** | `KRX_HOLIDAYS` 목록에 의해 스캔 Job SKIP |

### 9.2 주문 관련 예외

#### 미체결 주문

> 시장가 주문은 거의 즉시 체결되지만, 상한가/하한가 또는 유동성 부족 시 미체결 가능

**대응:**
```
매수 주문 후:
  1. 주문 실행 → 주문번호 반환
  2. 다음 스캔 시 체결 여부 확인 (일별체결조회 API)
  3. 미체결 상태이면:
     - 매수: 자동 취소 후 다음 스캔에서 재판단
     - 매도 (손절): 취소하지 않고 유지. 체결될 때까지 대기
```

#### 부분 체결

> 10주 매수 주문했는데 7주만 체결된 경우

**대응:**
```
- 부분 체결도 정상 체결로 취급
- 잔고 조회 시 실제 체결된 수량 기준으로 보유 수량 관리
- 미체결 잔량은 자동 취소 (매수) 또는 유지 (매도/손절)
```

#### 중복 주문 방지

> 5분 간격 스캔에서 같은 종목에 대해 매수 신호가 연속으로 발생

**대응:**
```
매수 전 체크:
  (1) 보유 종목 목록에 있는가? → SKIP (이미 다루고 있음)
  (2) 미체결 매수 주문이 존재하는가? → SKIP (새로 추가)

매도 전 체크:
  (1) 해당 종목에 미체결 매도 주문이 이미 있는가? → SKIP
```

#### 슬리피지 (시장가 주문)

> 시장가 주문 시 예상한 가격과 실제 체결 가격이 다를 수 있음

**대응:**
```
- 스윙 트레이딩은 수일~수주 보유이므로 소폭 슬리피지는 허용
- 체결 후 실제 체결가를 로그에 기록 (예상가 vs 실제가 비교)
- 추후 슬리피지가 문제되면 지정가 주문으로 전환 (설정으로 관리)
```

### 9.3 데이터 관련 예외

#### 일봉 데이터 부족

> 신규 상장 종목이거나, 거래일이 20일 미만인 경우 MA(20) 계산 불가

**대응:**
```
조회된 일봉 데이터 수 < MA_LONG_PERIOD(20) → 해당 종목 매매 SKIP
로그: "종목 {code}: 일봉 데이터 부족 ({N}일 < {MA_LONG_PERIOD}일)"
```

#### 비정상 시세 데이터

> API가 현재가 0원, 거래량 -1 등 비정상 데이터를 반환하는 경우

**대응:**
```
데이터 유효성 검증:
  - 현재가 <= 0 → SKIP
  - 거래량 < 0 → SKIP
  - 종가 배열에 0 또는 None 포함 → 해당 일자 제외 후 MA 재계산
  - 검증 실패 시 로그 기록 + 해당 종목 SKIP
```

#### API 응답 지연 / 타임아웃

**대응:**
```
httpx 타임아웃 설정 (10초)
  - 타임아웃 발생 → 해당 API 호출 실패 처리
  - 시세 조회 실패 → 해당 종목 이번 스캔 SKIP (다음 스캔에서 재시도)
  - 주문 API 실패 → 재시도 1회. 그래도 실패 시 SKIP + 로그
```

### 9.4 시스템 관련 예외

#### KIS API 서버 장애

**대응:**
```
연속 API 실패 횟수 카운트:
  - 3회 연속 실패 → 30초 대기 후 재시도
  - 5회 연속 실패 → 해당 스캔 전체 SKIP + 경고 로그
  - 10회 연속 실패 → 봇 전체 정지 + error.log 경고 (관리자 수동 확인 필요)
  - 성공 시 실패 카운트 초기화
```

#### 토큰 만료 (스캔 도중)

**대응:**
```
API 호출 → EGW00123 (토큰 만료) 응답 수신
  → KisAuthRepository.get_token() 재호출 (자동 갱신)
  → 새 토큰으로 원래 API 재호출 (최대 1회)
  → 재시도에도 실패 → 해당 스캔 SKIP
```

#### 앱 비정상 종료 후 재시작

> 프로세스가 죽었다가 다시 시작된 경우

**대응:**
```
앱 시작 시:
  1. 토큰 새로 발급 (이전 토큰은 사용 불가할 수 있음)
  2. 보유 종목 잔고 조회 (KIS API로 실제 잔고 동기화)
  3. 미체결 주문 조회 → 미체결 매수 주문은 전부 취소
  4. 정상 스케줄링 재개

주의: 앱의 메모리 상태(어떤 종목을 보유 중인지)에 의존하지 않고,
매 스캔마다 KIS API로 실제 잔고를 조회하여 상태를 동기화한다.
```

#### 스케줄러 지연/누락

> 시스템 부하로 5분 간격 스캔이 밀려서 10분 뒤에 실행되는 경우

**대응:**
```
- 스캔 시작 시 현재 시간 확인
- 15:30 이후면 → 스캔 SKIP (장 마감)
- 15:20 이후면 → 매도만 실행, 매수 금지
- 그 외에는 정상 실행 (시간 지연 자체는 스윙 전략에 큰 영향 없음)
```

## 10. Strategy 인터페이스 설계 (확장 포인트)

현재는 골든크로스/데드크로스 단일 전략이지만, 추후 전략을 교체하거나 조합할 수 있도록 인터페이스를 설계한다.

```
Protocol: TradingStrategy
  │
  ├── evaluate_buy(stock_code, market_data) → BuySignal | None
  │     입력: 종목코드, 시세 데이터
  │     출력: 매수 신호 (수량, 가격 포함) 또는 None
  │
  ├── evaluate_sell(stock_code, position, market_data) → SellSignal | None
  │     입력: 종목코드, 보유 포지션, 시세 데이터
  │     출력: 매도 신호 (수량, 사유 포함) 또는 None
  │
  └── get_required_data_days() → int
        출력: 전략 계산에 필요한 최소 일봉 일수 (MA20이면 25일)
```

**구현 예시 (추후):**

```
MaCrossStrategy          ← 현재 기본 전략 (골든/데드크로스)
BollingerBandStrategy    ← 볼린저 밴드 전략 (확장)
RsiStrategy              ← RSI 과매수/과매도 전략 (확장)
CompositeStrategy        ← 여러 전략 조합 (확장)
```

```
│                                                        │
│  3. 보유 종목 매도 체크 (종목별 순회)                      │
│     ├→ 시세 조회 실패? → 해당 종목 SKIP                  │
│     ├→ 현재가 <= 0 or 비정상? → SKIP                    │
│     ├→ 전일 대비 |변동률| > 30%? → 매매 보류             │
│     ├→ 미체결 매도 주문 이미 있음? → SKIP                 │
│     ├→ [P0] 손절: 수익률 <= -5%? → 즉시 매도             │
│     ├→ [P1] 익절: 수익률 >= +15%? → 매도                 │
│     ├→ [P2] 트레일링스탑: 고점 -4%? → 매도               │
│     ├→ [P3] 보유기간: 20일 초과? → 매도                   │
│     └→ [P4] 데드크로스: 2일 확인 후 → 매도                │
│                                                        │
│  4. 관심 종목 매수 체크 (종목별 순회)                      │
│     ├→ [사전] 시장 필터: KOSPI < MA(20)? → 전체 매수 중단  │
│     ├→ [사전] 일일 매수 횟수 >= 3? → 매수 중단             │
│     ├→ 안전성 필터 (관리종목/이상변동/상한가 등)            │
│     ├→ 품질 필터 (최소가격/거래대금/거래량/시총)            │
│     ├→ 이미 보유 중? / 미체결 매수? / 보유한도? → SKIP      │
│     ├→ 재매수 쿨다운 24시간 체크 → SKIP                   │
│     ├→ 일봉 데이터 조회 (60일) + 유효성 검증                │
│     ├→ MA 계산 + 골든크로스/상승추세 판별 (14일 내 교차 + 3일 유지)   │
│     ├→ ★ 거래량 확인: 교차일 거래량 >= 평균 × 1.5? → SKIP  │
│     ├→ ★ RSI 필터: RSI(14) 30~70 범위? → SKIP            │
│     ├→ 현재가 > MA(20)? + MA(20) 상승 추세?              │
│     ├→ 매수 수량 계산 (투자비율 10%, 수수료 반영)           │
│     └→ 수량 >= 1? → 매수 실행                             │
│                                                        │
│  5. 실행 결과 로깅                                       │
│     ├→ 매수/매도 실행 내역                                │
│     ├→ SKIP된 종목과 사유                                 │
│     └→ API 실패/예외 발생 기록                            │
│                                                        │
└────────────────────────────────────────────────────────┘
```

## 12. 시장 상황 필터 (Market Regime Filter)

> **하락장에서 골든크로스를 믿고 매수하면 큰 손실**을 입는다.
> KOSPI 지수 자체가 하락 추세일 때는 개별 종목의 골든크로스도 가짜 신호일 확률이 높다.

### 12.1 필터 원리

```
KOSPI 현재가 < KOSPI MA(20)?
  → YES: 하락장 판정 → 신규 매수 전면 중단
  → NO: 정상 → 매수 로직 진행
```

| 설정 | 기본값 | 설명 |
|------|--------|------|
| `ENABLE_MARKET_FILTER` | `true` | 시장 필터 활성화 여부 |

### 12.2 왜 효과적인가

한국 주식시장의 역사적 데이터를 보면:
- **상승장**에서 골든크로스 승률: ~55-60%
- **하락장**에서 골든크로스 승률: ~25-35%
- 시장 필터만 적용해도 **가짜 신호의 40% 이상을 사전 차단** 가능

### 12.3 구현 시점

매수 판단(`_evaluate_buy`) 초반부에 적용.
안전성 필터 통과 후, 품질 필터 전에 체크한다.
KOSPI 데이터도 TTL 캐시(5초)를 사용하므로 종목별 중복 호출 없음.

## 13. 매매 비용 분석 (수수료 + 세금)

### 13.1 한국 주식 거래 비용 구조

| 비용 항목 | 매수 시 | 매도 시 | 비고 |
|-----------|---------|---------|------|
| 매매 수수료 | 0.015% | 0.015% | 한국투자증권 온라인 기준 |
| 증권거래세 | - | 0.15% | 코스피/코스닥 동일 (2025~) |
| 슬리피지 (추정) | 0.05~0.15% | 0.05~0.15% | 대형주 기준. 시장가 주문 시 체결가 불리 |
| **소계** | **0.08~0.17%** | **0.25~0.35%** | |

### 13.2 왕복 비용 (Round-trip Cost) — 슬리피지 포함

```
[명시적 비용]
  수수료 왕복 + 거래세 = 0.015% + 0.015% + 0.18% = 0.21%

[슬리피지 (시장가 주문의 현실)]
  매수: 호가 스프레드 + 체결 지연 ≈ 0.05~0.15%
  매도: 특히 손절 시 급락 중 체결 → 0.1~0.3%
  평균 슬리피지 ≈ 0.15% (왕복)

[실제 왕복 비용]
  = 0.21% + 0.15% = 약 0.36%
```

> **매수 후 즉시 매도하면 -0.36% 손실.** 문서에 쓰던 0.21%보다 현실은 **1.7배** 비쌈.

### 13.3 매도 시나리오별 순수익 (슬리피지 포함)

| 시나리오 | 수익률(세전) | 실제 비용 | 순수익률 | 판정 |
|----------|-------------|-----------|----------|------|
| 손절 | -5.0% | -0.46% (급락 슬리피지 크) | **-5.46%** | 손실 |
| 트레일링스탑 (평균) | +7.0% | -0.36% | **+6.64%** | 이익 |
| 익절 | +15.0% | -0.36% | **+14.64%** | 이익 |
| 데드크로스 (조기 청산) | +2.0% | -0.36% | **+1.64%** | 얇은 이익 |
| 보유기간 초과 (횡보) | ±0% | -0.36% | **-0.36%** | 소손실 |

### 13.4 기대수익 계산 — 현실적 버전

> **핵심 변수:** 거래량 확인 + RSI 필터 추가로 승률이 어떻게 변하는가

**기존 (MA 크로스오버만):**

```
승률: ~40%, 평균 이익: ~+4.9% (대부분 얇은 청산), 평균 손실: -5.5%
EV = 0.40 × 4.9% - 0.50 × 5.5% - 0.10 × 0.36%
   = 1.96% - 2.75% - 0.04% = -0.83% / 매매  ← 마이너스
```

**개선 후 (거래량 확인 + RSI 필터 추가):**

> 거래량이 수반된 골든크로스만 매수하면 허약한 신호가 걸러져
> ① 승률이 10~15%p 상승하고 ② 평균 이익도 높아진다 (추세 지속력 증가).

```
[보수적 추정]
  승률: 50% (기존 40% + 거래량/RSI 필터로 +10%p)
  평균 이익: +6.0% (허약한 +1~2% 청산이 줄어듦)
  평균 손실: -5.5%
  무승부: 10%

  EV = 0.50 × 6.0% - 0.40 × 5.5% - 0.10 × 0.36%
     = 3.00% - 2.20% - 0.04% = +0.76% / 매매

[현실적 추정 (마찰 30% 할인)]
  조정 EV = +0.76% × 0.7 = +0.53% / 매매
  월 6~8회 매매 (거래량 필터로 매매 횟수 감소)
  월간 기대수익 = +0.53% × 7회 = +3.7%
```

**비관적 추정 (승률 45%):**

```
  EV = 0.45 × 6.0% - 0.45 × 5.5% - 0.10 × 0.36%
     = 2.70% - 2.475% - 0.036% = +0.19% / 매매
  마찰 할인 후: +0.13% / 매매
  월간: +0.13% × 7 = +0.9%  (거의 손익분기, 하지만 플러스)
```

### 13.5 손익분기 승률 (보수적 2-결과 모델)

> 아래 계산은 "무승부(횡보 청산)"를 패배로 간주한 보수적 2-결과 모델 기준.
> §13.4의 3-결과 모델보다 엄격한 기준을 적용하여 안전마진을 확보한다.

```
평균 이익 +6.0%, 평균 손실 -5.5%, 비용 -0.36% 기준:

  Win% × 6.0% - (1 - Win%) × 5.5% = 0.36%
  Win% × 11.5% = 5.86%
  Win% ≈ 50.9%

→ 거래량/RSI 필터 없이 순수 MA만으로는 50.9% 필요 (달성 어려움)
→ 거래량/RSI 필터 추가 시 50%+ 기대 가능 (손익분기 근처~소폭 이익)
```

### 13.6 솔직한 평가

> **이 전략은 "확실히 돈을 버는 시스템"이 아니다.**
> 거래량/RSI 필터를 추가해도 기대수익은 **월 +1~4% 범위**이며,
> 시장 환경에 따라 마이너스가 될 수 있다.
>
> 하지만:
> - 시장 필터로 **하락장의 대량 손실은 방지**된다
> - 리스크 관리로 **최악의 경우에도 자산의 5% 이내 손실**로 제한된다
> - 모의투자에서 **실제 데이터를 축적**하면 전략 개선의 기반이 된다
>
> **결론: "돈을 버는 봇"이 아니라 "안전하게 학습하고 검증하는 시스템"이다.**
> 모의투자 → 데이터 축적 → 전략 개선 → 소액 실전의 순서를 반드시 지킨다.

## 14. 손실 시나리오 & 대응 전략

### 14.1 시나리오 1: 갭다운 (Gap Down) — 손절선 관통

> 전날 종가 대비 -5% 이상 갭다운으로 시작.
> 시장가 매도해도 실제 체결가는 -7~10%일 수 있다.

| 구분 | 내용 |
|------|------|
| 발생 확률 | 연 2~5회 (대형주 기준, 어닝쇼크/악재) |
| 최대 손실 | -10% (1종목 기준) |
| 완화 | 종목당 투자 비율 10% 제한 → 전체 자산 -1% |
| 대응 | MIN_MARKET_CAP 5000억 필터로 대형주만 거래 → 갭다운 폭 제한 |

### 14.2 시나리오 2: 횡보장 — 잦은 거짓 신호

> KOSPI가 박스권에서 횡보하면 골든크로스/데드크로스가 반복적으로 발생.
> 매수 → 손절 반복으로 수수료만 소모.

| 구분 | 내용 |
|------|------|
| 발생 확률 | 연 3~4개월 |
| 최대 손실 | ~-0.36% × 매매 횟수 (수수료+슬리피지) + 잦은 -5% 손절 |
| 완화 | 시장필터(KOSPI < MA20 → 매수 중단) + 3일 확인 기간으로 대부분 필터링 |
| 대응 | 일일 매수 3회 제한 + 재매수 쿨다운 24h → 과도한 매매 방지 |

### 14.3 시나리오 3: 연속 하락장 (Bear Market)

> KOSPI가 수개월간 하락. 모든 매수 신호가 가짜.

| 구분 | 내용 |
|------|------|
| 발생 확률 | 2~3년에 1회 (예: 2022 하반기) |
| 최대 손실 | 시장필터 없으면 -20~30% |
| 완화 | **시장필터가 핵심 방어**. KOSPI < MA(20) 동안 신규 매수 전면 중단 |
| 대응 | 보유 중 종목은 손절/트레일링스탑으로 청산. 현금 비중 자동 증가 |

### 14.4 시나리오 4: 특정 종목 리스크 (뉴스/실적)

> 관심 종목이 갑자기 관리종목 지정, 실적 쇼크, 규제 뉴스.

| 구분 | 내용 |
|------|------|
| 발생 확률 | 수시 |
| 완화 | 안전성 필터(관리종목/투자유의/거래정지 실시간 체크) |
| 대응 | 이미 보유 중인 경우 → 손절 라인으로 손실 제한. 매수 전이면 → 필터에서 차단 |

### 14.5 시나리오 5: API/시스템 장애

> KIS API 장애, 네트워크 단절, 서버 다운.

| 구분 | 내용 |
|------|------|
| 발생 확률 | 연 수회 (짧은 장애) |
| 최대 손실 | 매도 지연으로 인한 추가 하락 |
| 완화 | MAX_CONSECUTIVE_FAILURES=5 → 자동 스캔 중단 + 경고 로그 |
| 대응 | 손절 매도 실패 시 1회 자동 재시도. FastAPI 수동 주문 엔드포인트로 긴급 매도 가능 |

### 14.6 최악 시나리오 (Maximum Drawdown)

```
5종목 동시 보유, 모두 갭다운 -10% → 전체 손실:
  종목당 투자 = 자산 × 10% = 100만원 (자산 1000만원 기준)
  종목당 손실 = 100만원 × -10% = -10만원
  총 손실 = -10만원 × 5종목 = -50만원 = 자산의 -5%

→ 한 번의 최악 상황에서도 전체 자산의 5%만 손실
→ 복구에 약 2~3개월 소요 (월 +2% 가정)
```

### 14.7 연간 시뮬레이션 (보수적)

```
[전제 — 현실적 추정]
- 초기 자산: 1,000만원 (모의투자)
- 월 평균 매매: 7회 (거래량/RSI 필터로 횟수 감소)
- 승률: 50% (거래량+RSI 필터 적용)
- 평균 이익: +6.0%, 평균 손실: -5.5% (슬리피지 포함)
- 1회 EV: +0.53% (마찰 할인 후)
- 월간 기대수익: +3.7%

[12개월 복리 시뮬레이션 — 보수적]
- 3월: 1,000 × 1.037^3 = ~1,115만원
- 6월: 1,000 × 1.037^6 = ~1,243만원
- 12월: 1,000 × 1.037^12 = ~1,545만원 (+54.5%)

[비관적 시나리오: 승률 45%]
- 1회 EV: +0.13% (마찰 할인 후)
- 월간: +0.9%
- 12월: 1,000 × 1.009^12 = ~1,113만원 (+11.3%)

[최악 시나리오: 승률 40% — 손익분기 미달]
- 1회 EV: -0.39%
  = 0.40 × 6.0% - 0.50 × 5.5% - 0.10 × 0.36%
  = 2.40% - 2.75% - 0.04% = -0.39%
- 월간: -0.39% × 7 = -2.7%
- 12월: 1,000 × 0.973^12 = ~720만원 (-28.0%)
- → 시장필터가 하락장 매수를 차단하므로 실제 손실은 이보다 작음
```

> **핵심:** 이 전략은 "확정 수익"이 아니다.
> - 승률 50%+ → 연 +50% 가능 (낙관적이지만 비현실적이진 않음)
> - 승률 45% → 연 +11% (은행 이자보다 나은 수준)
> - 승률 40% 미만 → 손실 발생 → 전략 교체 필요
>
> **모의투자 3개월 후 실측 승률로 판단해야 한다.**
> 실측 승률 45% 미만이면 전략을 교체하거나 보조지표를 추가한다.

## 15. 전략의 한계와 솔직한 평가

### 15.1 이 전략이 잘 작동하는 환경

- KOSPI가 우상향 또는 완만한 상승 추세
- 개별 대형주에 뚜렷한 트렌드가 형성될 때
- 변동성이 적당한 시장 (VIX 15~25 수준)

### 15.2 이 전략이 취약한 환경

- 급등·급락이 반복되는 고변동성 장세
- 뉴스 주도 시장 (실적, 정책 등이 기술적 분석을 압도)
- 장기 횡보장 (골든크로스 자체가 발생하지 않음)

### 15.3 전략으로 커버할 수 없는 영역

| 한계 | 설명 | 향후 대응 |
|------|------|----------|
| 급락 갭다운 | 장 시작 전 발생 → 대응 불가 | 분산 투자로 영향 최소화 |
| 상장폐지 | 극히 드물지만 발생 가능 | MIN_MARKET_CAP 필터 (대형주만) |
| 세금 제도 변경 | 금융투자소득세 등 | .env에서 세율 변경으로 즉시 대응 |
| 장중 급변 | 5분 간격으로만 감지 | 향후 WebSocket 실시간 대응 확장 가능 |

## 16. 운영자 행동 규칙 (심리적 개입 방지)

> 봇은 감정이 없지만, 전원 버튼을 쥔 운영자는 감정이 있다.
> 아래 규칙을 **사전에 서약**하고, 위반하면 봇을 즉시 중단한다.

### 16.1 절대 금지 행위

| 번호 | 금지 행위 | 이유 |
|------|----------|------|
| 1 | 봇의 매도 신호를 수동 취소 | "이건 더 오를 것 같아" = 가장 흔한 손실 원인 |
| 2 | 봇이 SKIP한 종목을 수동 매수 | 필터를 무시하면 필터의 의미가 없음 |
| 3 | 손실 중에 봇 중단 후 수동 매매 | 바닥에서 패닉 매도 → 최악의 결과 |
| 4 | 모의투자 결과 없이 실전 전환 | Phase 5 최소 3개월 필수 |
| 5 | 잃으면 안 되는 돈으로 실전 투자 | 실전 자산 = 전액 잃어도 생활에 지장 없는 금액 |

### 16.2 봇 정지 기준 (자동 + 수동)

```
[자동 정지 — 봇이 스스로 판단]
  - 연속 API 실패 5회 → 해당 스캔 SKIP (이미 구현)
  - 연속 API 실패 10회 → 봇 전체 정지 + error.log 경고

[수동 정지 — 운영자가 판단 (이 조건에서만 봇을 끌 수 있다)]
  - 모의투자 1개월 누적 승률 < 35% → 전략 재검토
  - 실전 계좌 MDD -15% 도달 → 봇 정지 + 보유 종목 수동 점검
  - KIS API 장기 장애 (1시간+) → 봇 정지
  - 전략 로직 버그 발견 → 봇 정지 + 수정 후 재배포
```

### 16.3 봇 재가동 규칙

```
정지 후 재가동 조건:
  1. 정지 원인이 해결되었는가? (버그 수정 / API 복구)
  2. 최소 1영업일 대기 (감정적 재가동 방지)
  3. 모의투자로 먼저 1주일 테스트
  4. 문제없으면 실전 복귀

예외: KIS API 단순 장애 → 복구 확인 후 즉시 재가동 가능
```

### 16.4 주간 리뷰 의무

```
매주 금요일 장 마감 후:
  1. daily_reports 확인 → 주간 승률, 수익률, MDD
  2. 손절 종목 분석 → 왜 신호가 잘못됐는지 (기록용)
  3. SKIP 사유 분석 → 필터가 과도하게 차단하고 있지 않은지
  4. 결론 기록 → "파라미터 변경 필요" 또는 "유지"
  
  ※ 파라미터 변경은 주간 리뷰에서만 가능.
    스캔 중간에 .env를 수정하지 않는다.
```
