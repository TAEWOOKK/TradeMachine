# 매매 전략 규칙서 — 단타 (Scalping)

> TradeMachine의 핵심인 규칙 기반(Rule-based) **단타 전략**을 정의한 문서.
> 오를 것 같은 주식을 사서 빠르게 익절(+2%)하고, 손절(-2%)로 손실을 제한한다.
>
> **변경 이력**: [strategy-changelog.md](./strategy-changelog.md) 참조.

---

## 1. 전략 개요

| 항목 | 값 |
|------|-----|
| 전략 유형 | 규칙 기반 **단타** (Scalping) |
| 대상 시장 | 국내주식 (코스피/코스닥) |
| 보유 기간 | **당일~2영업일** (빠른 회전) |
| 목표 | 오르는 종목 진입 → **익절(+2%)** 우선, **손절(-2%)**로 리스크 관리 |
| 핵심 지표 | 일봉 MA + **전일 대비 등락률**(오르는 중 감지) |
| 판단 주기 | 스케줄러 간격 (기본 5분) |
| 소수점 매매 | 사용하지 않음 (정수 단위만) |

---

## 2. 단타의 핵심 원리

### 2.1 "오르는 주식"에 올라타기

```
단타 목표:
  → 전일 종가 대비 이미 +0.3%~+4% 상승한 종목 (설정값 기준)
  → MA(5) > MA(20)로 상승 추세 확인
  → 추가 상승(+2%) 가능성 있는 구간에 진입
  → 익절(+2%) 또는 손절(-2%)로 빠르게 청산
```

### 2.2 장 초반 변동성 회피

```
09:00~09:25: 변동성 최대 → 매수 보류
09:25~15:20: 변동 완화 후 매수 허용
```

| 설정 | 기본값 | 설명 |
|------|--------|------|
| `SCALPING_ENTRY_MINUTE` | 25 | 09:XX (25분) 이후부터 매수 허용. 0이면 제한 없음 |

### 2.3 데이터 소스

- **현재가 시세** (`FHKST01010100`): 현재가, **전일 종가 대비 등락률**(`prdy_ctrt`), 거래량, 거래대금
- **기간별 시세** (`FHKST03010100`): 일봉 → MA(5), MA(20), RSI 계산

### 2.4 전일 대비 등락률의 의미 (한계)

- `prdy_ctrt`는 **전일 종가 대비 현재가** 비율이다. **당일 분봉 추세나 장중 저점 대비 반등**을 직접 보지는 않는다.
- “단타”라는 **보유·청산 타이밍**과 맞추었지만, **진입 신호는 일봉 지표 + 전일대비 혼합**이다. 장중 분봉 기반 단타로 바꾸려면 API·로직 확장이 필요하다.

### 2.5 장마감 청산(EOD)과 최대 보유일

- `EOD_CLOSE_ENABLED=true`이면 **매 영업일 15:28**에 보유를 비우므로, 통상 **야간 이월 보유가 없다.**
- `MAX_HOLDING_DAYS`는 **EOD가 꺼져 있거나**, **15:28 매도 실패로 다음날까지 물린 경우** 등 예외에 대비한 **안전망**으로 둔다.

## 3. 매수 조건 (진입 규칙)

### 3.1 필수 조건 (모두 충족)

```
[1] 전일 대비 등락률 (change_rate)
    MIN_INTRADAY_CHANGE <= change_rate <= MAX_INTRADAY_CHANGE
    → "오르는 중"이지만 과열 전 구간
    기본: +0.3% ~ +4.0%

[2] 상승 추세 (일봉 MA)
    MA(5) > MA(20) (최근 3일 연속 유지)
    현재가 >= MA(5)

[3] RSI 필터
    47 <= RSI <= 65
    → 모멘텀 있으나 과열 아님

[4] 진입 시간
    장 시작 후 SCALPING_ENTRY_MINUTE(25)분 경과
    → 09:25 이후 매수
```

| 설정 | 기본값 | 설명 |
|------|--------|------|
| `MIN_INTRADAY_CHANGE` | 0.3 (%) | 전일 대비 최소 상승률 (오르는 종목만) |
| `MAX_INTRADAY_CHANGE` | 4.0 (%) | 전일 대비 최대 상승률 (과열 구간 제외) |
| `SCALPING_ENTRY_MINUTE` | 25 | 장 시작 후 N분 이후 매수 허용 (0=제한없음) |
| `RSI_SCALPING_MIN` | 47 | 단타용 RSI 하한 |
| `RSI_SCALPING_MAX` | 65 | 단타용 RSI 상한 |

### 3.2 매수 판별 플로우

```
관심 종목 순회
  │
  ▼
[중복/한도 체크] (기존과 동일)
  │
  ▼
[진입 시간 체크]
  현재 시각 < 09:SCALPING_ENTRY_MINUTE? → SKIP "장 초반 매수 보류"
  │
  ▼
[현재가 조회] (change_rate 포함)
  │
  ▼
[전일 종가 대비 등락률 체크]
  change_rate < 0? → SKIP "하락 중 진입 금지"
  change_rate < MIN_INTRADAY_CHANGE? → SKIP "상승 미흡"
  change_rate > MAX_INTRADAY_CHANGE? → SKIP "이미 과열"
  │
  ▼
[종목 품질 필터] (기존: 주가, 거래대금, 거래량, 시총)
  │
  ▼
[일봉 조회 + MA/RSI 계산]
  │
  ▼
[상승 추세 체크]
  MA(5) > MA(20) 3일 연속?  현재가 >= MA(5)? → NO → SKIP
  │
  ▼
[RSI 체크]
  RSI 47~65? → NO → SKIP
  │
  ▼
★ 매수 실행 ★ (OrderReason: SCALPING_ENTRY)
```

### 3.3 매수 수량 결정

```
1. 매수 가능 금액 조회 (KIS API)
2. 종목당 투자 금액 = 매수 가능 금액 / n_slots (기존과 동일)
3. 매수 수량 = 투자 금액 ÷ (현재가 × 1.00015) (소수점 버림)
4. 1주 이상이면 매수
```

### 3.4 매수 주문 방식

| 항목 | 값 | 이유 |
|------|-----|------|
| 주문 유형 | **시장가** | 단타는 즉시 체결이 핵심 |

## 4. 매도 조건 (청산 규칙)

매도는 우선순위 순으로 실행한다.

### 4.1 손절 (P0, 최우선)

```
현재 수익률 <= STOP_LOSS_RATE → 즉시 매도
```

| 설정 | 기본값 | 설명 |
|------|--------|------|
| `STOP_LOSS_RATE` | -2.0 (%) | 매입가 대비 2% 하락 시 손절 |

### 4.2 익절 (P1)

```
현재 수익률 >= TAKE_PROFIT_RATE → 매도
```

| 설정 | 기본값 | 설명 |
|------|--------|------|
| `TAKE_PROFIT_RATE` | 2.0 (%) | 매입가 대비 2% 상승 시 익절 |

### 4.3 트레일링 스탑 (P2)

```
[1단계: 활성화]
  보유 중 수익률이 TRAILING_STOP_ACTIVATE(+1.5%) 이상 도달한 적 있음

[2단계: 매도]
  현재가 <= 최고가 × (1 - TRAILING_STOP_RATE / 100) → 매도
```

| 설정 | 기본값 | 설명 |
|------|--------|------|
| `TRAILING_STOP_ACTIVATE` | 1.5 (%) | +1.5% 도달 시 트레일링 활성화 |
| `TRAILING_STOP_RATE` | 0.8 (%) | 고점 대비 0.8% 하락 시 매도 |

### 4.4 보유 기간 초과 (P3)

```
보유 영업일 > MAX_HOLDING_DAYS → 매도
```

| 설정 | 기본값 | 설명 |
|------|--------|------|
| `MAX_HOLDING_DAYS` | 2 | 단타는 2영업일 초과 시 강제 청산 |

### 4.5 데드크로스 (P4)

```
MA(5)가 MA(20) 아래로 돌파 + 2일 유지 → 매도
```

### 4.6 장마감 전량 청산 (매 영업일)

```
월~금 15:28 + EOD_CLOSE_ENABLED → 보유 전량 매도
(야간·익일 시가 갭 리스크 방지. 금요일만이 아님.)
```

### 4.7 매도 판별 플로우 (우선순위)

```
P0 손절:    수익률 <= -2%     → 즉시 매도
P1 익절:    수익률 >= +2%     → 매도
P2 트레일링: +1.5% 도달 후 -0.8% 하락 → 매도
P3 기간초과: 보유 2영업일 초과 → 매도
P4 데드크로스: MA(5) < MA(20) 2일 유지 → 매도
```

## 5. 리스크 관리

### 5.1 단타용 기본값

| 항목 | 값 | 설명 |
|------|-----|------|
| 손절 | -2% | 타이트한 손실 제한 |
| 익절 | +2% | 빠른 수익 실현 |
| 트레일링 활성화 | +1.5% | 단타는 낮은 구간에서 활성화 |
| 트레일링 하락 | 0.8% | 고점 대비 소폭 하락 시 매도 |
| 최대 보유 | 2영업일 | 단타 회전율 |

### 5.2 일일 한도

| 규칙 | 값 |
|------|-----|
| 일일 최대 매수 | 3회 |
| 재매수 쿨다운 | 24시간 (동일 종목) |
